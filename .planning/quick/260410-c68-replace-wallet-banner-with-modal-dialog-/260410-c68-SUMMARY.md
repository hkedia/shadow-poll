# Quick Task 260410-c68: Replace Wallet Banner with Modal Dialog Summary

## One-Liner
Transformed WalletOnboarding from a sticky banner (showing for not_detected, disconnected, error) to a centered modal dialog that only auto-appears for error states (not_detected, error), while disconnected state now relies on the header "Connect Wallet" button.

## What Changed

### Components

| File | Change |
|------|--------|
| `components/wallet-onboarding.tsx` | Complete rewrite: replaced banner markup with Radix UI Dialog modal |

### Behavior Matrix

| Status | Old Banner | New Modal |
|--------|------------|-----------|
| `idle` | Hidden | Hidden |
| `not_detected` | Banner shown | **Modal shown** |
| `disconnected` | Banner shown | **Hidden** (use header button) |
| `connecting` | Hidden | Hidden |
| `connected` | Hidden | Hidden |
| `error` | Banner shown | **Modal shown** |

## Implementation Details

### Modal Design
- **Overlay**: Dark backdrop `bg-black/60` with fade animation
- **Card**: Centered, `max-w-md`, `rounded-2xl`, `bg-surface-container-low`, shadow-2xl
- **Close**: X button in top-right corner, also dismisses on Escape key or outside click
- **Animation**: Radix UI default animations (fade, zoom)

### State Management
- `ModalStatus` type reduced to `not_detected | error` (removed `disconnected`)
- `MODAL_STATUSES` array excludes `disconnected`
- sessionStorage dismissal persists per status key (`wallet-modal-dismissed-${status}`)

### Content Preserved
- `not_detected`: Wallet icon, "Install 1am.xyz" heading, description, install CTA button
- `error`: Error icon, "Connection Failed" heading, error message, troubleshooting tips (collapsible), "Try Again" button

### Accessibility
- Uses Radix UI Dialog primitives for proper focus management
- `Dialog.Title` and `Dialog.Description` for screen readers
- `aria-label` on close button
