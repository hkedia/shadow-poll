```typescript
/**
 * Poll creation fee configuration for Shadow Poll.
 *
 * POLL_CREATION_FEE: Number of NIGHT tokens required to create a poll.
 * Change this constant to update the fee without touching business logic.
 *
 * getFeeTokenId: Returns the correct token type string for the given network.
 * On Preview and preprod: "tNIGHT" (testnet token)
 * On mainnet: "NIGHT" (production token)
 *
 * The token ID string must match what the Midnight runtime expects as the
 * shielded token denomination. The contract uses pad(32, tokenId) to produce
 * a Bytes<32> token identifier in the Compact receive() call.
 */

/** Number of NIGHT / tNIGHT tokens required to create a poll. */
export const POLL_CREATION_FEE: bigint = BigInt(5);

/** Preview testnet network ID as returned by the 1am wallet api.getConfiguration(). */
export const PREVIEW_NETWORK_ID = "preview";

/** Preprod network ID. */
export const PREPROD_NETWORK_ID = "preprod";

/** Mainnet network ID. */
export const MAINNET_NETWORK_ID = "mainnet";

/**
 * Returns the canonical token denomination string for the given network.
 *
 * On testnet networks (preview, preprod): "tNIGHT"
 * On mainnet: "NIGHT"
 *
 * This string is used both in UI display ("5 tNIGHT") and as the token ID
 * passed to the Compact contract's receive() call (via pad(32, tokenId)).
 *
 * @param networkId - The networkId from wallet api.getConfiguration()
 */
export function getFeeTokenId(networkId: string): string {
  if (networkId === MAINNET_NETWORK_ID) {
    return "NIGHT";
  }
  // Preview, preprod, and any unknown testnet network → tNIGHT
  return "tNIGHT";
}

/**
 * Returns a human-readable fee description for the given network.
 * Used by the UI fee banner.
 *
 * @example "Creating this poll costs 5 tNIGHT"
 */
export function getFeeBannerText(networkId: string): string {
  const tokenId = getFeeTokenId(networkId);
  return `Creating this poll costs ${POLL_CREATION_FEE.toString()} ${tokenId}`;
}

/**
 * Returns the insufficient balance error message for the given network.
 *
 * @example "Insufficient tNIGHT balance. You need 5 tNIGHT to create a poll."
 */
export function getInsufficientBalanceMessage(networkId: string): string {
  const tokenId = getFeeTokenId(networkId);
  return `Insufficient ${tokenId} balance. You need ${POLL_CREATION_FEE.toString()} ${tokenId} to create a poll.`;
}
```

Notes:
- Named exports, no default export (project convention).
- `POLL_CREATION_FEE` is `BigInt(5)` — matches Compact's `Uint<64>` type.
- The fallback in `getFeeTokenId` is testnet (`tNIGHT`) — safest default for Preview network.
- `getFeeBannerText` and `getInsufficientBalanceMessage` are thin helpers so the UI never formats strings itself.

  <name>Task 3: Extend create_poll Compact circuit with fee collection via receive()</name>
  <files>contracts/src/poll.compact</files>
  <action>
Add a shielded token fee transfer to the `create_poll` circuit in `contracts/src/poll.compact`.

Midnight Compact v0.19+ exposes `receive(tokenId: Bytes<32>, amount: Uint<64>)` as a built-in — it transfers the specified token from the circuit caller's shielded balance to the contract's own shielded balance. If the caller has insufficient balance, the circuit fails and the transaction is rejected on-chain.

**What to add:**

1. Add two new ledger declarations below the existing ledger state block (after `vote_nullifiers`):

```compact
// Fee configuration
// POLL_CREATION_FEE: number of tNIGHT/NIGHT tokens required to create a poll.
// Change this constant to update the fee. Stored as a Uint<64> for Compact compatibility.
export ledger poll_creation_fee: Uint<64>;

// Collected fee revenue (total NIGHT tokens received by the contract from poll creation).
// Informational — allows querying accumulated fees from the indexer.
export ledger total_fees_collected: Counter;
```

Wait — Compact ledger state for simple constants is unnecessary and complicates initialization. Instead, define the fee as a Compact module-level constant:

```compact
// Poll creation fee: 5 tNIGHT (or NIGHT on mainnet).
// To change the fee, update this constant and recompile/redeploy the contract.
const POLL_CREATION_FEE: Uint<64> = 5;
```

2. Add the token ID as a parameter to `create_poll` so it is disclosed (the token denomination used is a matter of public record):

Actually, per D-82, the token type is determined by the network at runtime. The token ID needs to match the shielded token denomination. In Compact, `pad(32, "tNIGHT")` produces the Bytes<32> token ID. However the token ID string must match the actual denomination on the network being used.

The cleanest approach for v1 (Preview network only): accept `token_id` as a parameter to `create_poll`, disclosed on-chain. The TypeScript service layer passes the correct token ID based on `getFeeTokenId(networkId)`.

**Updated create_poll circuit signature:**
```compact
export circuit create_poll(
  metadata_hash: Bytes<32>,
  option_count: Uint<8>,
  poll_type: PollType,
  expiration_block: Uint<64>,
  fee_token_id: Bytes<32>
): Bytes<32> {
```

**Add at the beginning of the circuit body (before existing validation):**
```compact
  // Disclose fee token — the denomination used is public information
  const d_fee_token_id = disclose(fee_token_id);

  // Collect poll creation fee from the caller's shielded balance (D-82).
  // Fails atomically if the caller has insufficient token balance.
  receive<QualifiedCoinInfo>(d_fee_token_id, POLL_CREATION_FEE);
```

Add `const POLL_CREATION_FEE: Uint<64> = 5;` as a module-level constant before the ledger declarations.

The full change to `contracts/src/poll.compact`:

**Before the ledger declarations block, add:**
```compact
// --- Fee Configuration -------------------------------------------------------

// Poll creation fee in tNIGHT / NIGHT tokens.
// Changing this constant requires contract redeployment.

```

**In the `create_poll` circuit signature, add `fee_token_id: Bytes<32>` parameter.**

**In the `create_poll` circuit body, after the existing parameter disclosures and before the option count validation, add:**
```compact
  // --- Fee collection ---------------------------------------------------------
  const d_fee_token_id = disclose(fee_token_id);
  // Transfer POLL_CREATION_FEE tokens from the caller to this contract (D-82).
  // The runtime fails this circuit (and the whole transaction) if the caller
  // has insufficient tNIGHT/NIGHT balance — insufficient funds = rejected tx.
  receive<QualifiedCoinInfo>(d_fee_token_id, POLL_CREATION_FEE);
```

Note on Compact `receive` syntax: Compact v0.19 uses `receive<CoinType>(tokenId, amount)`. The `QualifiedCoinInfo` type is the standard coin type from `CompactStandardLibrary`. If the compiler rejects `QualifiedCoinInfo`, try `CoinInfo` or just `receive(d_fee_token_id, POLL_CREATION_FEE)` without a type parameter — check compiler output and fix accordingly (Rule 1 auto-fix).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → create_poll circuit | Caller provides fee_token_id. Circuit discloses it on-chain. Insufficient balance causes atomic tx failure — no partial state. |
| fee-config.ts → UI/service | POLL_CREATION_FEE is a compile-time constant — cannot be tampered with at runtime |
| networkId → token selector | networkId comes from wallet configuration (trusted 1am API). Attacker cannot fake mainnet to get charged tNIGHT instead of NIGHT |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | create_poll fee | mitigate | receive() is a Compact built-in — atomic with the tx. Caller cannot bypass it or underpay. |
| T-08-02 | Spoofing | fee_token_id parameter | accept | Caller could pass a wrong token ID (e.g., a worthless token). The Compact runtime validates that the caller has sufficient balance of the specified token. If the fee is paid with a different token, the contract accepts it — v1 does not validate the token ID against a whitelist. Future: add an assertion that d_fee_token_id == pad(32, "tNIGHT"). |
| T-08-03 | Denial of Service | fee barrier | accept | The fee deters spam poll creation. DUST for voting is still sponsored by 1am. |
| T-08-04 | Information Disclosure | networkId in providerSet | accept | networkId is not secret — it is the network name ("preview"). No PII. |
</threat_model>
