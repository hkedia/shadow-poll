<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user form → createPoll mutation | User title/options → on-chain + metadata API |
| URL params → usePoll | Untrusted poll ID from URL → indexer query |
| indexer → results panel | Tally data from indexer → displayed percentages |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Tampering | create-poll-form.tsx (input) | mitigate | Client-side validation via `validatePollMetadata()` before submission. Contract enforces option_count ≤ 10 on-chain. |
| T-04-10 | Tampering | poll/[id]/page.tsx (URL param) | mitigate | Poll ID from URL is hex string — validated by `hexToBytes()` which handles malformed input. usePoll returns null for non-existent polls. |
| T-04-11 | Information Disclosure | results-panel.tsx (tallies) | accept | Vote tallies are public by design (D-21). Only voter identity is private. |
| T-04-12 | Denial of Service | page.tsx (poll list) | accept | Poll list fetched from indexer with 30s staleTime. Indexer handles load. Client-side caching prevents excessive re-fetches. |
</threat_model>
