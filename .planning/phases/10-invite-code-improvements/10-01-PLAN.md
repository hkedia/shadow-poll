# Phase 10, Plan 1: Invite Code Contract + Service Fixes

## Goal

Fix two bugs in the invite-only poll system:
1. Batch invite code submission (1 transaction for N codes)
2. Single-use enforcement for invite codes

## Pre-conditions

- `contracts/src/poll.compact` compiles with Compact 0.30+
- `lib/midnight/contract-service.ts` imports from `@/contracts/managed/contract`
- All invite-only poll flows wire through `use-create-poll.ts` and `use-add-invite-codes.ts`

## Changes

### 1. `contracts/src/poll.compact` — Two circuit changes

**1a. Make `add_invite_codes` generic over the number of codes**

Replace the single-hash signature with a generic circuit that accepts a fixed-size vector:

```compact
// Before
export circuit add_invite_codes(
  poll_id: Bytes<32>,
  code_hash: Bytes<32>
): [] { ... }

// After
export circuit add_invite_codes<#N>(
  poll_id: Bytes<32>,
  code_hashes: Vector<N, Bytes<32>>
): [] {
  // ... same auth checks ...
  for (const code_hash of code_hashes) {
    const d_code_hash = disclose(code_hash);
    invite_codes.insert(d_code_hash, true);
  }
}
```

**1b. Mark invite codes as spent in `cast_invite_vote`**

After verifying the invite code hash is present, check that it's still valid (value == true) and then mark it as spent (value = false):

```compact
// Before — just membership check
assert(invite_codes.member(d_invite_key), "Invalid invite code");

// After — membership AND unused check, then mark spent
assert(invite_codes.member(d_invite_key), "Invalid invite code");
assert(invite_codes.lookup(d_invite_key), "Invite code already used");
invite_codes.insert(d_invite_key, false);
```

### 2. `contracts/managed/contract/index.d.ts` — Update type signature

The Compact compiler regenerates this file, but we also update it manually to unblock TypeScript before recompilation:

```typescript
// Before
add_invite_codes(context: ..., poll_id_0: Uint8Array, code_hash_0: Uint8Array): ...

// After
add_invite_codes(context: ..., poll_id_0: Uint8Array, code_hashes_0: Uint8Array[]): ...
```

### 3. `lib/midnight/contract-service.ts` — Update `AddInviteCodeParams` and `callAddInviteCodes`

```typescript
// Before
export interface AddInviteCodeParams {
  pollId: Uint8Array;
  codeHash: Uint8Array;
}

export async function callAddInviteCodes(contract, params: AddInviteCodeParams) {
  return await contract.callTx.add_invite_codes(params.pollId, params.codeHash);
}

// After
export interface AddInviteCodesParams {
  pollId: Uint8Array;
  codeHashes: Uint8Array[]; // All hashes submitted in one tx
}

export async function callAddInviteCodes(contract, params: AddInviteCodesParams) {
  return await contract.callTx.add_invite_codes(params.pollId, params.codeHashes);
}
```

### 4. `lib/queries/use-add-invite-codes.ts` — Single call

```typescript
// Before: loop
for (const codeHash of params.codeHashes) {
  await callAddInviteCodes(contract, { pollId, codeHash });
}

// After: single call
await callAddInviteCodes(contract, { pollId: pollIdBytes, codeHashes: params.codeHashes });
```

### 5. `lib/queries/use-create-poll.ts` — Single call at creation

```typescript
// Before: loop
for (const code of inviteCodes) {
  await callAddInviteCodes(contract, { pollId: pollIdBytes, codeHash: code.hash });
}

// After: single call
await callAddInviteCodes(contract, { pollId: pollIdBytes, codeHashes: inviteCodes.map(c => c.hash) });
```

## Success Criteria

1. Contract compiles with `bash contracts/scripts/compile.sh --skip-zk` without errors
2. `add_invite_codes` circuit accepts an array of hashes and registers them all in one transaction
3. `cast_invite_vote` circuit rejects a code that has already been used (value == false)
4. `callAddInviteCodes` TypeScript function submits all hashes in a single `contract.callTx` call
5. `useCreatePoll` and `useAddInviteCodesMutation` both use the new single-call interface
6. `bun run build` succeeds with no TypeScript errors

## Notes

- After changing the contract, run `bash contracts/scripts/compile.sh --skip-zk` to verify syntax. Full ZK recompilation with `bash contracts/scripts/compile.sh` is needed before deployment, but is slow and can be deferred.
- The `index.d.ts` update is manual until recompilation runs; ensure it matches the new vector interface.
- No UI changes are required — the invite code panel already shows all codes at once; the change is purely in how many transactions are submitted.
