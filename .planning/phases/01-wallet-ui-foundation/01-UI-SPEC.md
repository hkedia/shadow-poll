## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon and label |
| sm | 8px | Compact element spacing, pill internal padding |
| md | 16px | Default element spacing, card padding, button horizontal padding |
| lg | 24px | Section padding, header vertical padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Hero section vertical padding |
| 3xl | 64px | Page-level top/bottom spacing, hero section vertical margin |

Exceptions: Touch targets minimum 44px height on mobile (wallet connect button, hamburger menu icon button). This is not a spacing token вҖ” it's a minimum interactive size.

## Typography

All typography uses Geist Sans (`font-sans` / `var(--font-geist-sans)`) unless noted otherwise.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (1rem) | 400 (regular) | 1.5 (24px) | Paragraph text, descriptions, onboarding body copy |
| Label | 14px (0.875rem) | 500 (medium) | 1.4 (19.6px) | Button text, badge text, navigation items, form labels, truncated addresses |
| Heading | 24px (1.5rem) | 600 (semibold) | 1.2 (28.8px) | Card headings, section titles, onboarding step titles |
| Display | 36px (2.25rem) | 700 (bold) | 1.1 (39.6px) | Hero headline only |

**Monospace exception:** Wallet addresses use Geist Mono (`font-mono` / `var(--font-geist-mono)`) at 14px, weight 400, line-height 1.4. This provides visual distinction for hexadecimal content.

### 60/30/10 Split

| Split | Color | Elements |
|-------|-------|----------|
| 60% Dominant | `--background` (`#09090b`) | Page body, main content area background |
| 30% Secondary | `--card` / `--secondary` (`#0e0e12` / `#27272a`) | Header bar, cards, wallet dropdown, mobile drawer, onboarding overlay |
| 10% Accent | `--primary` (`#7c3aed` violet) | See reserved list below |

### Accent Reserved For (exhaustive list)

1. "Connect Wallet" CTA button (hero + header)
2. Focus rings on all interactive elements
3. Active/selected navigation state
4. Step indicator dots in onboarding flow (active step)
5. Loading spinner stroke color
6. Link hover underlines
7. Wallet status "connected" indicator dot

**Accent is NOT used for:** body text, card backgrounds, borders, icons at rest, disabled states, or decorative elements.

### Onboarding Flow (Guided Steps)

Three-step overlay:

1. **Step 1: Detect** вҖ” Checking for 1am.xyz extension (auto-advances)
2. **Step 2: Install** (conditional) вҖ” Shows install prompt if not detected
3. **Step 3: Connect** вҖ” "Connect your wallet" with connect button

- **Container:** Full-viewport overlay with `bg-background/80 backdrop-blur-sm`
- **Card:** Centered, max-width 480px, `--card` background, 24px padding
- **Step indicator:** Row of dots, 8px each, active dot uses `--primary`, inactive uses `--muted`
- **Transitions:** 300ms ease-in-out slide between steps

## CSS Variables Implementation Note

The `globals.css` file must be updated to replace the current light/dark media-query approach with dark-only CSS variables. The `<html>` element must have `class="dark"` hardcoded.
