**Change 2: use-poll.ts — two-path poll query (unauthenticated + authenticated)**

Rewrite the `pollQuery` `useQuery` inside `usePoll()`. The tallies query remains wallet-gated and unchanged.

Replace the `pollQuery` block (lines 37–50) with:

```typescript
const pollQuery = useQuery({
  queryKey: pollKeys.detail(pollId ?? ""),
  queryFn: async (): Promise<PollWithId | null> => {
    if (!pollId) return null;

    // Unauthenticated path: fetch via server API (no wallet required)
    // Per D-09-06/07: remove isConnected gate for read queries
    const contractAddress = getContractAddress();

    if (!providers || !contractAddress) {
      // No wallet — use server API (returns data from indexer without wallet)
      const res = await fetch(`/api/polls?id=${encodeURIComponent(pollId)}`);
      if (!res.ok) return null;
      const data = await res.json();
      const p = data.poll;
      if (!p) return null;

      // Map server response to PollWithId shape
      const { hexToBytes } = await import("@/lib/midnight/ledger-utils");
      const { PollType } = await import("@/contracts/managed/contract");
      const pollIdBytes = hexToBytes(p.id);
      return {
        id: p.id,
        idBytes: pollIdBytes,
        data: {
          metadata_hash: hexToBytes(p.metadataHash),
          option_count: BigInt(p.optionCount),
          poll_type: p.pollType === 1 ? PollType.invite_only : PollType.public_poll,
          expiration_block: BigInt(p.expirationBlock),
          creator: hexToBytes(p.creator),
        },
      };
    }

    // Authenticated path: use live indexer via fetchPollWithTallies (wallet connected)
    const result = await fetchPollWithTallies(providers, contractAddress, pollId);
    return result?.poll ?? null;
  },
  enabled: !!pollId,  // Removed isConnected gate (per D-09-06)
  refetchInterval: 15_000,
});
```

Keep the tallies query (lines 53–66) exactly as-is — tallies remain wallet-gated since they require live indexer reads. Update `enabled` in the tallies query from `isConnected && !!pollId && pollQuery.data !== null` to `isConnected && !!pollId` to avoid the circular dependency with `pollQuery.data !== null` (it was never null when isConnected, just redundant).

**Important:** The `isConnected` variable at line 33 is still used by the tallies query — keep it. Only remove it from the poll query's `enabled` field.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| public internet → GET /api/polls | No auth — all poll data is public; indexer is public |
| public internet → GET /api/polls/metadata | No auth — metadata is public (off-chain supplement to on-chain hash) |
| public internet → POST /api/polls/metadata | No auth — metadata hash comparison enforces integrity |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-06 | Info Disclosure | GET /api/polls (no auth) | accept | All poll data is public by design — on-chain state is public; no PII in polls |
| T-09-07 | Tampering | POST /api/polls/metadata — false metadata injection | mitigate | Already mitigated: metadata-handler.ts recomputes SHA-256 hash of incoming metadata; rejects if mismatch with submitted metadataHash. On-chain hash is authoritative. |
| T-09-08 | Spoofing | usePoll() unauthenticated path trusts server response | accept | Server reads from Midnight indexer (canonical source); no client-supplied state is trusted for on-chain data |
| T-09-09 | Denial of Service | GET /api/polls loads all polls every single-poll request | accept | Acceptable for v1 poll counts; can add ?id= server filter (implemented in this plan) to reduce payload |
</threat_model>
