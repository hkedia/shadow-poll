## Directory Purposes

**`app/`:**
- Purpose: Next.js App Router root — all routes, layouts, pages, and (future) API routes
- Contains: `.tsx` page/layout components, `.css` stylesheets, static assets like `favicon.ico`
- Key files: `layout.tsx` (root layout), `page.tsx` (home page), `globals.css` (global styles)

**`public/`:**
- Purpose: Static file serving — files are accessible at `/{filename}` in the browser
- Contains: SVG icons and logos (scaffold defaults)
- Key files: `next.svg`, `vercel.svg`

**`.planning/`:**
- Purpose: Project planning documentation and codebase analysis
- Contains: `codebase/` subdirectory with architecture mapping documents
- Key files: Generated by codebase mapping tooling

## Key File Locations

**Entry Points:**
- `app/layout.tsx`: Root layout — wraps every page, defines `<html>` and `<body>`, loads Geist fonts
- `app/page.tsx`: Home page — renders at `/`

**Configuration:**
- `package.json`: Dependencies, scripts (`dev`, `build`, `start`, `lint`)
- `next.config.ts`: Next.js configuration (currently default/empty)

- `eslint.config.mjs`: ESLint flat config with `eslint-config-next` core-web-vitals and TypeScript presets
- `postcss.config.mjs`: PostCSS with Tailwind CSS v4 plugin

**Core Logic:**
- None yet — project is at scaffold stage

**Testing:**
- No test files, no test configuration, no test framework installed

## Naming Conventions

**Files:**
- Pages: `page.tsx` (Next.js App Router convention)
- Layouts: `layout.tsx` (Next.js App Router convention)
- Global styles: `globals.css`
- Config files: `{name}.config.{ts,mjs}` (e.g., `next.config.ts`, `eslint.config.mjs`, `postcss.config.mjs`)

**Directories:**
- Route segments: lowercase (e.g., `app/`)
- Static assets: `public/`

## Where to Add New Code

**New Page Route:**
- Create: `app/{route-segment}/page.tsx`
- Example: `app/poll/[id]/page.tsx` for a dynamic poll page

**New Layout (nested):**
- Create: `app/{route-segment}/layout.tsx`
- Wraps all pages under that route segment

**New API Route:**
- Create: `app/api/{endpoint}/route.ts`
- Export named functions: `GET`, `POST`, `PUT`, `DELETE`
- GraphQL endpoint (planned): `app/api/graphql/route.ts`

**New Component:**
- No convention established yet. Recommended: `app/_components/` or a top-level `components/` directory
- Use `"use client"` directive only when client interactivity is needed

**New Utility/Library Code:**
- No convention established yet. Recommended: `lib/` at project root
- Midnight SDK integration logic: `lib/midnight/` or similar

**New Server Actions:**
- Define in separate files with `"use server"` directive at top
- Recommended: `app/_actions/` or co-located with the page that uses them

**Static Assets:**
- Place in `public/` — accessible at `/{filename}` URL

**Tests:**
- No test infrastructure exists yet. When added, co-locate as `{name}.test.tsx` or use a `__tests__/` directory

## Import Patterns

**Path Aliases:**
- `@/*` maps to `./*` (project root) — configured in `tsconfig.json`
- Use `@/app/...`, `@/lib/...`, etc. for absolute imports

**Current Import Style (from existing files):**
- Framework imports first: `import type { Metadata } from "next"`
- Package imports: `import { Geist, Geist_Mono } from "next/font/google"`
- Local/relative imports last: `import "./globals.css"`

**Barrel Exports:**
- Not used (no `index.ts` barrel files exist)
