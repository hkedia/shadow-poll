# Quick Task 260410-c68: Replace wallet banner with modal dialog for error states - Context

**Gathered:** 2026-04-10
**Status:** Ready for planning

<domain>
## Task Boundary

Replace the current dismissible app-wide banner (WalletOnboarding component) with a modal dialog system for wallet error states. The current banner shows below the header and handles `not_detected`, `disconnected`, and `error` states. The new implementation should:

1. Remove the banner entirely
2. Show a modal dialog only for error states (`not_detected`, `error`)
3. Keep the Connect Wallet button in the header (already exists)
4. Auto-trigger modal when wallet detection fails or network error occurs
5. Allow users to dismiss the modal (session-only persistence)
6. Support "connecting" state in the header button (already implemented)

The `disconnected` state should NOT show a modal — users should explicitly click "Connect Wallet" to initiate connection.

</domain>

<decisions>
## Implementation Decisions

### Modal Trigger States
- **Decision:** Modal shows only for error states: `not_detected` and `error`
- **Rationale:** `disconnected` is a neutral state — user hasn't initiated connection yet
- **Excludes:** `idle`, `disconnected`, `connecting`, `connected`

### Auto-Trigger Behavior
- **Decision:** Modal auto-triggers automatically when wallet detection fails or network is wrong
- **Implementation:** Check status on mount and when status transitions to error states
- **No trigger:** Manual click on "Connect Wallet" should NOT show modal unless error occurs

### Dismissal Persistence
- **Decision:** Session-only dismissal (same as current banner)
- **Implementation:** Use `sessionStorage` with key `wallet-modal-dismissed-${status}`
- **Reset:** Modal reappears on page refresh for same error state

### Agent's Discretion
- Modal design and styling (follow existing design system patterns)
- Error content/messaging for each state
- Whether to use existing Dialog component or create new
- Animation/transition behavior

</decisions>
