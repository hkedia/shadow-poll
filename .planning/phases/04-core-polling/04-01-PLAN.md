The factory function `createWitnesses<PS>(secretKey: Uint8Array, blockNumber: bigint)` returns an object matching the `Witnesses<PS>` type from the compiled contract:

- `local_secret_key(context)`: Returns `[context.privateState, secretKey]` where `secretKey` is the 32-byte key passed in. The secret key comes from the wallet's shielded address (derived at call time).
- `current_block_number(context)`: Returns `[context.privateState, blockNumber]` where `blockNumber` is the current network block number passed in.

Both must return the exact tuple `[PS, ReturnType]` â€” the first element is ALWAYS `context.privateState` unchanged.

Export the factory function and a helper `getSecretKeyFromWallet(enabledApi: any): Promise<Uint8Array>` that:
  1. Calls `enabledApi.getShieldedAddresses()` to get the shielded address info
  2. Uses the `shieldedCoinPublicKey` (or similar 32-byte key) as the secret key input
  3. Falls back to hashing the shielded address string to Uint8Array(32) if the wallet doesn't expose a raw key â€” use `crypto.subtle.digest("SHA-256", new TextEncoder().encode(address))` and return new Uint8Array of the result
  4. Important: This is a deterministic derivation â€” same wallet always produces same key

Also export `getCurrentBlockNumber(indexerUri: string): Promise<bigint>` that queries the indexer for the latest block height. Use a simple GraphQL query via fetch to the indexer URI:
```graphql
{ blockHeight { latest } }
```
If the query fails or the indexer doesn't expose this, fall back to `BigInt(0)` with a console.warn (the ZK proof will still be generated, the prover just needs a plausible block number â€” the contract's assert verifies it).

```typescript
export async function assembleMidnightProviders(providerSet: MidnightProviderSet) {
  // 1. Create real SDK PublicDataProvider via createIndexerProvider()
  const { createIndexerProvider } = await import("./indexer");
  const publicDataProvider = await createIndexerProvider(
    providerSet.indexerConfig.indexerUri,
    providerSet.indexerConfig.indexerWsUri,

  return {
    publicDataProvider,
    zkConfigProvider: providerSet.zkConfigProvider,
    proofProvider: providerSet.proofProvider,
    walletProvider: providerSet.walletProvider,
    midnightProvider: providerSet.walletProvider, // wallet API also handles tx submission
    privateStateProvider: {
      // In-memory private state â€” sufficient for testnet (assumption A5)
      get: async () => undefined,
      set: async () => {},
    },
  };
}
```

**Contract address management strategy (per research open question 1):**
Use environment variable `NEXT_PUBLIC_POLL_CONTRACT_ADDRESS`. If set, `findPollContract` uses it. If empty, `deployPollContract` deploys a new instance and returns the address. The app decides at runtime: if env var is set, use find; otherwise, deploy on first use.

**Exports:**

1. `deployPollContract(providers, witnesses)` â€” Deploys a fresh Poll Manager contract.
   - Dynamic import `@midnight-ntwrk/midnight-js-contracts` for `deployContract`
   - Dynamic import `@midnight-ntwrk/compact-js` for `CompiledContract`
   - Dynamic import `@/contracts/managed/contract` for `Contract`
   - Build the compiled contract: Use `new Contract(witnesses)` â€” the SDK's `deployContract` needs the contract instance with witnesses attached
   - Call `deployContract(providers, { contract: compiledContract })` â€” check the actual SDK signature from node_modules since the research examples use a builder API that may differ from the installed version
   - Return the deployed contract object (which has `.callTx` for circuit calls and `.deployTxData` for the address)

2. `findPollContract(providers, contractAddress, witnesses)` â€” Finds an already-deployed contract.
   - Same dynamic imports as deploy
   - Call `findDeployedContract(providers, { contract: compiledContract, contractAddress })`
   - Return the found contract object

3. `callCreatePoll(contract, params)` â€” Calls the `create_poll` circuit.
   - `params`: `{ metadataHash: Uint8Array, optionCount: number, pollType: PollType, expirationBlock: bigint }`
   - Calls `contract.callTx.create_poll(metadataHash, BigInt(optionCount), pollType, expirationBlock)`
   - Returns the result (which contains the poll ID as `Uint8Array`)

4. `callCastVote(contract, params)` â€” Calls the `cast_vote` circuit.
   - `params`: `{ pollId: Uint8Array, optionIndex: number }`
   - Calls `contract.callTx.cast_vote(pollId, BigInt(optionIndex))`
   - Returns void (cast_vote returns `[]`)

5. `getContractAddress()` â€” Returns the canonical contract address.
   - Reads `process.env.NEXT_PUBLIC_POLL_CONTRACT_ADDRESS`
   - Returns `string | null`

6. `fetchAllPolls(publicDataProvider, contractAddress)` â€” Queries contract state for all polls.
   - Gets contract state via `publicDataProvider.queryContractState(contractAddress)`
   - Parses with dynamic import of `ledger` from `@/contracts/managed/contract`
   - Iterates `ledgerState.polls` via `Symbol.iterator` (as documented in contract/index.d.ts)
   - Returns `PollWithId[]` (from ledger-utils.ts)

7. `fetchPollWithTallies(publicDataProvider, contractAddress, pollIdHex)` â€” Fetches a single poll + tallies.
   - Gets contract state, parses ledger

   - Uses `readTallies(ledger, hexToBytes(pollIdHex), optionCount)` from ledger-utils
   - Returns `{ poll: PollWithId, tallies: PollTallies } | null`
