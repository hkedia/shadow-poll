Create directory `contracts/src/` and write `contracts/src/poll.compact`.

**Compact Syntax Rules (CRITICAL — from AGENTS.md and skills):**
- Pragma: `pragma language_version >= 0.19;`
- Ledger: Individual `export ledger` declarations — NOT `ledger { }` block syntax
- Return type: `[]` (empty tuple) — NOT `Void`
- Witness: Declaration only — NO body in Compact (implementation goes in TypeScript prover)
- Pure helpers: `pure circuit` — NOT `pure function`
- Enum access: Dot notation (e.g., `PollType.public`) — NOT double colon (`PollType::public`)
- Counter read: `counter.read()` — NOT `counter.value()`
- Assertions: `assert(condition, "message")`
- Disclosure: Wrap witness-derived comparisons in `disclose()` for conditionals

**Contract structure per D-20 through D-24:**

```compact
pragma language_version >= 0.19;

import CompactStandardLibrary;
```

**Enum for poll type (per D-21 — public polls for Phase 2):**
```compact
export enum PollType { public_poll, invite_only }
```

**Struct for poll data stored on-chain (per D-22 — hash-based metadata):**
```compact
export struct PollData {
  metadata_hash: Bytes<32>,
  option_count: Uint<8>,
  poll_type: PollType,
  expiration_block: Uint<64>,
  creator: Bytes<32>,
}
```

**Ledger state — individual declarations (per D-20, D-23):**
- `export ledger polls: Map<Bytes<32>, PollData>;` — poll registry mapping poll IDs to poll data
- `export ledger tallies: Map<Bytes<32>, Vector<10, Counter>>;` — per-poll vote tallies (Vector<10, Counter> per D-23)
- `export ledger poll_count: Counter;` — total number of polls created

**Witness declarations (no body):**
- `witness local_secret_key(): Bytes<32>;` — caller's secret key for identity derivation

**Helper circuits:**
- `pure circuit derive_public_key(sk: Bytes<32>): Bytes<32>` — derives a public key from secret key using `persistentHash<Vector<2, Bytes<32>>>([pad(32, "shadowpoll:pk:"), sk])`
- `pure circuit derive_poll_id(metadata_hash: Bytes<32>, creator: Bytes<32>, count_bytes: Bytes<32>): Bytes<32>` — derives a unique poll ID from metadata hash + creator + poll count using `persistentHash<Vector<3, Bytes<32>>>([metadata_hash, creator, count_bytes])`

**Circuit: `create_poll` (per CONT-01, D-22, D-23, D-24):**
```
export circuit create_poll(
  metadata_hash: Bytes<32>,
  option_count: Uint<8>,
  poll_type: PollType,
  expiration_block: Uint<64>
): Bytes<32>
```
- Assert `option_count > 0` and `option_count <= 10` ("Option count must be 1-10")
- Get caller public key via `derive_public_key(local_secret_key())`
- Read current poll count via `poll_count.read()` and cast to `Bytes<32>` for hashing
- Derive poll_id via `derive_poll_id(metadata_hash, caller_pk, count_as_bytes)`
- Create PollData struct with all fields
- Insert into `polls` map: `polls.insert(poll_id, poll_data)`
- Insert default tallies: `tallies.insertDefault(poll_id)`
- Increment poll_count: `poll_count.increment(1)`
- `disclose()` the poll_id and return it

**Circuit: `cast_vote` (per CONT-02, D-24):**
```
export circuit cast_vote(
  poll_id: Bytes<32>,
  option_index: Uint<8>
): []
```
- Assert poll exists: `assert(polls.member(poll_id), "Poll not found")`
- Look up poll data: `const poll_data = polls.lookup(poll_id)`
- Assert option_index is valid: `assert(option_index < poll_data.option_count, "Invalid option index")`
- NOTE: Block-number-based expiration check (D-24) — Compact does NOT have a built-in `block_number()` function. Add a comment: `// TODO: Expiration enforcement requires block_number access — will be enforced via witness in Phase 4 when SDK integration is wired`. For now, add a witness declaration `witness current_block_number(): Uint<64>;` and assert `current_block_number() < poll_data.expiration_block` with message "Poll has expired". This enforces expiration client-side (the TypeScript prover provides the real block number). The ZK proof guarantees the voter cannot lie about the block number because the proof would be invalid if the assertion fails.
- Look up tallies vector for this poll and increment the counter at `option_index`
  - NOTE: Vector element access in Compact uses index syntax. The tallies for a poll are `tallies.lookup(poll_id)` which returns `Vector<10, Counter>`. Incrementing the correct counter requires index access. If direct index access on Vector<Counter> is not supported by the compiler, use a conditional chain pattern: `if (option_index == 0 as Uint<8>) { ... } else if (option_index == 1 as Uint<8>) { ... }` etc. for all 10 options. This is a known Compact pattern for fixed-size vectors.
- Do NOT store the voter's wallet address or public key alongside their choice (per D-21 — private voter identity). The ZK proof validates the vote is legitimate without linking voter to option.

**Important: Do NOT use these incorrect patterns:**
- ❌ `ledger { ... }` block syntax
- ❌ `Void` return type
- ❌ `pure function` keyword
- ❌ `PollType::public_poll` double-colon access
- ❌ `counter.value()` — use `counter.read()`
- ❌ Witness with implementation body
