# Plan: Replace Wallet Banner with Modal Dialog for Error States

## Goal
Replace the dismissible sticky banner with a centered modal dialog that only appears for wallet error states (`not_detected`, `error`), while keeping the header Connect button as the entry point for the `disconnected` state.

## Task 1: Transform WalletOnboarding to Modal
**Files:** `components/wallet-onboarding.tsx`

**Action:**
Replace the sticky banner implementation with a centered modal dialog using Radix UI Dialog primitive:

1. Import `@radix-ui/react-dialog` components (already available via sheet.tsx shows it's installed)
2. Change modal trigger states: Only show for `not_detected` and `error` — remove `disconnected` from the list
3. Create modal layout with:
   - Dark overlay backdrop (`bg-black/60`)
   - Centered modal card with `bg-surface-container-low`, `rounded-2xl`, `shadow-2xl`
   - Max-width `max-w-md` for readability
4. Keep existing content sections:
   - `not_detected`: Show wallet icon, "Install 1am.xyz" heading, description, and primary CTA button linking to 1am.xyz
   - `error`: Show error icon, "Connection Failed" heading, error message, troubleshooting tips (collapsible), and "Try Again" button
5. Keep sessionStorage dismissal logic with key `wallet-modal-dismissed-${status}`
6. Add close button (X) in top-right corner of modal
7. Use Aether design system colors (CSS custom properties like `--surface-container-low`, `--primary`, etc.)

**Verify:**
- Component renders no visible output when status is `idle`, `disconnected`, `connecting`, or `connected`
- Modal appears automatically when status becomes `not_detected` or `error`
- Modal content matches the design system (lavender primary, navy-black backgrounds)
- Dismissing the modal sets sessionStorage and hides it until page refresh

**Done:**
- `wallet-onboarding.tsx` contains modal markup (no banner HTML)
- Modal only triggers for `not_detected` and `error` states
- Existing content and functionality preserved (icons, messaging, CTA buttons, troubleshooting)
- sessionStorage dismissal works per status key

## Task 3: Test Modal Flows
**Files:** (manual verification)

**Action:**
Verify the three wallet states behave correctly:

1. **Disconnected state**: 
   - Load page without wallet extension or when extension not detected yet
   - Status becomes `disconnected`
   - NO modal should appear
   - User clicks "Connect Wallet" button in header
   - If wallet not found, status becomes `not_detected` → modal appears

2. **Not detected state**:
   - Trigger by loading page without 1am.xyz extension installed
   - Modal should auto-appear
   - Click "Install 1am.xyz" → opens in new tab
   - Click X to dismiss → modal closes, doesn't reappear until refresh

3. **Error state**:
   - Trigger by rejecting connection in wallet or wrong network
   - Modal should auto-appear with error message
   - Troubleshooting section should expand/collapse
   - "Try Again" button should re-trigger connection flow
   - Dismissal persists per session

**Verify:**
- `bun run dev` starts successfully
- No TypeScript errors in modified files
- All three flows work as expected in browser

**Done:**
- All wallet states behave correctly
- Modal appears only for error states
- Dismissal persistence works
- Connect button remains functional for disconnected flow

## Implementation Notes

### Modal vs Banner Behavior Matrix
| Status | Old Banner | New Modal |
|--------|------------|-----------|
| `idle` | Hidden | Hidden |
| `not_detected` | Banner shown | **Modal shown** |
| `disconnected` | Banner shown | **Hidden** (use header button) |
| `connecting` | Hidden | Hidden |
| `connected` | Hidden | Hidden |
| `error` | Banner shown | **Modal shown** |

### Key Design Decisions
- Using Radix UI Dialog (via `@radix-ui/react-dialog`) for accessibility and focus management
- Same dismissal persistence pattern as banner (sessionStorage keyed by status)
- No changes to WalletButton — it continues to show "Connect Wallet" for disconnected state
- Modal overlays at `z-50` to appear above all content
