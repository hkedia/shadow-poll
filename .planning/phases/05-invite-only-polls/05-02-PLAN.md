Create a new file `lib/midnight/invite-codes.ts` — the off-chain invite code management utility.

This module handles:
- Generating random invite codes (alphanumeric strings)
- Converting codes to Bytes<32> for hashing
- Computing code hashes (for add_invite_codes circuit)
- Storing/loading codes in localStorage (D-53: browser-only storage)

**Exports:**

1. `generateInviteCodes(count: number, pollId: Uint8Array): Promise<InviteCodeSet>`

```typescript
export interface InviteCode {
  code: string;         // Human-readable code (e.g., "A3K9F2X7B1")
  codeBytes: Uint8Array; // Code as Bytes<32> (SHA-256 hash of the string)
  hash: Uint8Array;     // On-chain key = deriveInviteKey(pollId, codeBytes)
  hashHex: string;      // Hex-encoded hash for display
}

export interface InviteCodeSet {
  pollId: string;       // Hex-encoded poll ID
  codes: InviteCode[];  // Generated codes
  createdAt: string;    // ISO timestamp
}
```

Implementation:
- Generate `count` random alphanumeric codes, each 10 characters (A-Z, 0-9)
- Use `crypto.getRandomValues()` for secure randomness
- For each code string:
  a. Hash the string to Bytes<32> via SHA-256: `inviteCodeToBytes32(codeString)` — this normalizes the variable-length string to a fixed 32 bytes
  b. Compute the on-chain key: `deriveInviteKey(pollId, codeBytes)` from ledger-utils
- Return the full set

2. `inviteCodeToBytes32(code: string): Promise<Uint8Array>`

Converts a human-readable invite code string to Bytes<32> using SHA-256:
```typescript
export async function inviteCodeToBytes32(code: string): Promise<Uint8Array> {
  const encoded = new TextEncoder().encode(code.toUpperCase().trim());
  const hashBuffer = await crypto.subtle.digest("SHA-256", encoded);
  return new Uint8Array(hashBuffer);
}
```

Case-insensitive (uppercase before hashing) so users can type codes in any case.

3. `storeInviteCodes(pollIdHex: string, codes: InviteCode[]): void`

Stores invite codes in localStorage under `shadow-poll:invite-codes:{pollIdHex}`.
Serializes as JSON (converting Uint8Arrays to hex strings for storage).

4. `loadInviteCodes(pollIdHex: string): InviteCode[] | null`

Loads invite codes from localStorage. Returns null if not found.
Deserializes hex strings back to Uint8Arrays.

5. `formatCodesForCopy(codes: InviteCode[]): string`

Returns all codes as a newline-separated string for clipboard copy.

6. `formatCodesForCSV(codes: InviteCode[], pollTitle: string): string`

Returns codes as CSV content: `Code,Hash\nA3K9F2X7B1,abc123...\n...`

CRITICAL:
- Use `crypto.getRandomValues()` — NOT `Math.random()` — for code generation
- Use `crypto.subtle.digest("SHA-256", ...)` for inviteCodeToBytes32
- The code → Bytes<32> conversion MUST be deterministic: same code string → same bytes → same hash
- Uppercase normalization before hashing ensures case-insensitive code entry

  const result = await contract.callTx.cast_invite_vote(
    params.pollId,

}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user input → inviteCodeToBytes32 | User-typed invite code normalized and hashed |
| localStorage → invite-codes.ts | Codes stored/retrieved from browser storage |
| invite-codes.ts → contract | Generated code hashes submitted as on-chain txs |
| mutation → contract | All invite-related params flow into ZK circuit |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Information Disclosure | invite-codes.ts (localStorage) | accept | Codes stored in creator's browser only (D-53). If browser is compromised, codes are exposed — but this is a client-side security boundary. No server-side storage. |
| T-05-07 | Tampering | use-invite-vote-mutation.ts (code) | mitigate | Code is hashed via SHA-256 then verified by ZK circuit against on-chain Map. Invalid codes are rejected by the contract assertion. |
| T-05-08 | Denial of Service | use-add-invite-codes.ts (batch) | accept | Sequential tx submission for N codes. Creator pays gas for each. Natural rate limiting via blockchain fees. |
| T-05-09 | Spoofing | use-create-poll.ts (pollType) | mitigate | pollType parameter (0 or 1) is stored on-chain. Cannot be changed after creation. Contract enforces correct circuit usage per poll type. |
