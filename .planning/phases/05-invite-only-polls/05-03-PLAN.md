async function handleDownloadCSV() {

  const csv = formatCodesForCSV(codes, pollTitle);
  const blob = new Blob([csv], { type: "text/csv" });

  a.href = url;
  a.download = `shadow-poll-${pollId.slice(0, 8)}-codes.csv`;
  a.click();

}

| user form → pollType toggle | User selects poll privacy level |
| user input → invite code field | User-typed invite code sent to ZK circuit |
| clipboard → invite codes | Codes copied to clipboard (browser API) |
| localStorage → invite-codes.ts | Codes persisted in browser storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Spoofing | vote-panel.tsx (invite code input) | mitigate | Code is normalized (uppercase) before hashing. The ZK circuit verifies the hash against on-chain Map. Invalid codes are rejected by contract assertion. User sees clear error message. |
| T-05-11 | Information Disclosure | invite-code-panel.tsx (clipboard) | accept | Clipboard access requires HTTPS or localhost. Codes are visible in the browser — this is by design (creator must share them). Warning message reminds creator to save codes. |
| T-05-12 | Tampering | create-poll-form.tsx (pollType) | mitigate | pollType is stored on-chain at creation time and cannot be changed. The contract enforces correct circuit usage (cast_vote for public, cast_invite_vote for invite-only). |
| T-05-13 | Denial of Service | invite-code-panel.tsx (100 codes) | accept | Maximum 100 invite codes per poll. Each requires a separate on-chain tx to register. Creation may be slow for large code counts — UI shows progress. |
</threat_model>
