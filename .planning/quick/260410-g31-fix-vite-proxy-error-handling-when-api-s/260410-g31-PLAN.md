# Quick Task 260410-g31: Fix Vite Proxy Error Handling

**Date:** 2026-04-10
**Status:** Ready for execution

## Problem

When running only the Vite dev server (`bun run dev`) without the API server (`bun run dev:api`), the Vite proxy throws "socket hang up" errors for every `/api/*` request. This floods the console with error messages.

```
[vite] http proxy error: /api/polls/metadata?pollId=...
Error: socket hang up
```

## Solution

Update Vite proxy configuration with:
1. Error handling to suppress noisy error messages
2. Configure option to handle connection failures gracefully
3. Optionally add a bypass for health checks

## Files to Modify

1. `vite.config.ts` - Update proxy configuration with error handling

## Implementation

Replace simple proxy config with object-based configuration that includes:
- `target`: The API server URL
- `changeOrigin`: true
- `configure` function to add error event handler
- `rewrite` if needed

The key is adding an error handler to the proxy that either:
- Silently ignores connection errors (dev-only)
- Or logs a single warning instead of stack traces
