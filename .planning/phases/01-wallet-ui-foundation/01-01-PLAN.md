<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Google Fonts CDN | Font and icon CSS loaded from external CDN at runtime |
| User browser → App | All rendering is client-side; no user data flows in this plan |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Spoofing | Google Fonts CDN link | accept | Fonts are CSS-only (no executable); Next.js self-hosts Google fonts at build time for font files — Material Symbols loaded via link tag only for icon font, no user data exposed |
| T-01-02 | Information Disclosure | Material Symbols CDN request | accept | Icon font request reveals user visited the site — standard CDN risk, no PII, no wallet data. Mitigate in production by self-hosting icon font if privacy is paramount |
| T-01-03 | Tampering | globals.css CSS variables | accept | CSS vars are client-readable but carry no secrets; design tokens only |
| T-01-04 | Denial of Service | Excessive Tailwind class generation | accept | Tailwind v4 JIT is build-time only; no runtime CSS injection surface |
</threat_model>
