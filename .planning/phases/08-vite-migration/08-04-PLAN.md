<objective>
Fix the Vite 8 build failure caused by `vite-plugin-top-level-await@1` requiring the `rollup` module (which Vite 8 does not bundle — it uses Rolldown).

Purpose: Close the 2 verification gaps (build failure + WASM runtime) that share a single root cause — an incompatible Vite plugin. Output: Working `bun run build` and `bun run dev` commands.
</objective>

  <action>
    **Step 1: Remove the plugin from vite.config.ts**

    Edit `vite.config.ts`:
    - Remove the import line: `import topLevelAwait from "vite-plugin-top-level-await";`
    - Remove `topLevelAwait(),` from the `plugins` array
    - Keep all other plugins (`wasm()`, `react()`, `tailwindcss()`) and all other config unchanged

    The resulting plugins array should be:
    ```typescript
    plugins: [
      wasm(),
      react(),
      tailwindcss(),
    ],
    ```

    **Why safe to remove:** The plugin was added to handle top-level await in WASM modules, but:
    - Vite 8's Rolldown bundler natively supports top-level await when `build.target: "esnext"` (already configured)
    - `vite-plugin-wasm@3.6.0` handles WASM loading independently and supports Vite 8
    - No source files in `lib/`, `src/`, or `components/` use top-level await outside async functions
    - The plugin's `@rollup/plugin-virtual` dependency requires `rollup` which Vite 8 does not include

    **Step 2: Uninstall the package**

    Run: `bun remove vite-plugin-top-level-await`

    This removes the package from `package.json` dependencies and `node_modules/`.

    **Step 3: Verify the fix**

    Run: `bun run build`

    Expected: Vite build completes successfully, producing files in `dist/` directory.

    Run: `bun run dev` (start and immediately kill after confirming it starts)

    Expected: Vite dev server starts without "Cannot find module 'rollup'" error.
  </action>
  <acceptance_criteria>

    - vite.config.ts DOES contain "wasm()" in the plugins array
