# Quick Task 260410-g31 Summary

**Task:** Fix Vite proxy error handling when API server is not running
**Date:** 2026-04-10
**Status:** Complete

## Problem

When running only the Vite dev server (`bun run dev`) without the API server (`bun run dev:api`), the console was flooded with "socket hang up" errors:

```
[vite] http proxy error: /api/polls/metadata?pollId=...
Error: socket hang up
    at Socket.socketOnEnd (node:_http_client:617:25)
    ...
```

This happened for every API request when the backend wasn't running.

## Solution

Updated `vite.config.ts` proxy configuration with error handling:

**Before:**
```typescript
server: {
  proxy: {
    "/api": "http://localhost:3001",
  },
},
```

**After:**
```typescript
server: {
  proxy: {
    "/api": {
      target: "http://localhost:3001",
      changeOrigin: true,
      configure: (proxy, _options) => {
        proxy.on("error", (err, _req, _res) => {
          // Suppress ECONNREFUSED errors (API not running)
          if ((err as NodeJS.ErrnoException).code === "ECONNREFUSED") {
            return; // Silent fail
          }
          console.warn(`[vite-proxy] ${err.message}`);
        });
      },
    },
  },
},
```

## Changes

- Converted simple proxy string to object-based configuration
- Added `configure` callback to attach error event handler
- Silently ignore `ECONNREFUSED` errors (API server not running)
- Log other proxy errors with `[vite-proxy]` prefix for clarity
